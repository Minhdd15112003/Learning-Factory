## Nội dung mới hôm nay

### Khái niệm mới 1: ThreadPoolExecutor và RejectedExecutionHandler
- **Bounded Queue**: `newFixedThreadPool` dùng `LinkedBlockingQueue` unbounded → nguy cơ OOM. Tự tạo `ThreadPoolExecutor` với `LinkedBlockingQueue(capacity)` để giới hạn.
- **Sức chứa**: `maxPoolSize` + `queueCapacity`. Vượt quá → `RejectedExecutionHandler` xử lý.
- **4 policy**: AbortPolicy (ném exception — banking), CallerRunsPolicy (throttling tự nhiên), DiscardPolicy (vứt âm thầm — email), DiscardOldestPolicy (vứt cũ giữ mới).
- **Thực hành**: `ThreadPoolThrottlingPractice.java` — tạo pool coreSize=2, max=4, queue=5, submit 12 tasks. Quan sát AbortPolicy ném exception ở task 10, CallerRunsPolicy main thread tự chạy task.

### Khái niệm mới 2: CompletableFuture
- **Pipeline bất đồng bộ**: `supplyAsync` → `thenApply` → `thenAccept` — callback-based, không cần `get()` blocking.
- **Xử lý lỗi**: `exceptionally()` bắt lỗi → trả giá trị dự phòng → cứu pipeline.
- **Chạy song song**: `thenCombine()` gộp kết quả 2 future chạy đồng thời.
- **Phân biệt**: ThreadPoolExecutor = "Ai/Bao nhiêu" (resource). CompletableFuture = "Thế nào/Khi nào" (orchestration). Dùng chung qua `supplyAsync(() -> ..., executor)`.
- **Thực hành**: `CompletableFuturePractice.java` — 3 bài (pipeline, exceptionally, thenCombine ~1005ms).
- **Lỗi gặp phải**: Đặt `exceptionally()` sau `thenApply()` nhưng `supplyAsync` trả `Long`, lỗi nổ ở `exceptionally` trả `String` cho stage kiểu `Long`. Đã tự sửa.
