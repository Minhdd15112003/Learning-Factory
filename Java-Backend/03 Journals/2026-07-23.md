## Nội dung mới hôm nay

### Adapter Pattern
- **Vấn đề:** Tích hợp SDK bên ngoài (VNPay, MoMo) có interface không khớp với code nội bộ. Không được sửa SDK, sửa toàn bộ code nội bộ thì rải khắp nơi.
- **Cơ chế:** Adapter class implements interface nội bộ (`PaymentGateway`) + giữ object SDK bên ngoài qua constructor. Trong `charge()`: chuyển đổi tham số + bổ sung tham số thiếu từ field (constructor) + gọi method SDK.
- **Lỗi gặp:** `No enclosing instance` — inner class non-static dùng trong context static → cần thêm `static`.
- **Thực hành:** VnpayAdapter + MomoAdapter, gọi qua `processPayment(PaymentGateway gw, ...)` — code nội bộ không biết SDK nào đằng sau.

### Decorator Pattern
- **Vấn đề:** Nhét hành vi phụ (log, validate, audit) vào method gốc → method phình to, trộn logic, vi phạm Single Responsibility.
- **Cơ chế:** Decorator implements cùng interface + giữ `wrapped` (cùng interface) qua constructor. Trong method: thực hiện hành vi phụ → gọi `wrapped.execute()`. Lồng nhau như lớp vỏ hành — thứ tự lồng = thứ tự chạy.
- **Điểm hay:** ValidationDecorator chặn luồng bằng cách không gọi `wrapped.execute()`, nhưng LoggingDecorator bên ngoài vẫn chạy tiếp vì call stack trả về dòng tiếp theo sau `wrapped.execute()`.
- **Thực hành:** LoggingDecorator + ValidationDecorator + AuditDecorator lồng quanh CoreTransactionService.
